# Specifications

This directory contains lightweight specifications for features and changes.

## When to write a spec

| Change type | Spec? |
|---|---|
| Bug fix | No |
| Trivial change (< 1 day) | No |
| New feature, clear requirements | Optional |
| New feature, unclear requirements | Yes |
| Cross-cutting change | Yes |
| API design | Yes |

## Template

A spec is a 1-page markdown document covering:

```md
# {Feature Name}

## Problem
What problem are we solving? For whom?

## Success criteria
How will we know this is done? Measurable outcomes.

## Non-goals
What is explicitly out of scope?

## Open questions
What isn't decided yet?

## Sketch
Optional: rough approach, wireframes, or API shape.
```
